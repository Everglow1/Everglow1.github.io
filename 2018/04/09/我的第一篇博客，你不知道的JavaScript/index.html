Git介绍以及简单用法

发表于 2018-04-09 | 分类于 git | 阅读次数: |
Git是什么？

Git(读音为/gɪt/。)是一个开源的分布式版本控制系统，可以有效、高速的处理从小到非常大的项目版本管理。
Git是目前最流行的版本管理系统，学会Git几乎成为了开发者必备的技能

Git有很多优势，我觉得最方便的就是远程操作非常便捷，简单。 大概5个Git命令就能涵盖git远程操作

  1. git clone
  2. git remote
  3. git fetch
  4. git pull
  5. git push

git clone

远程操作的第一步就要用到git clone ，从远程主机克隆一个版本库, 会在自己本地主机生成一个目录，与远程主机的库同名

git clone <版本库的地址> 例如： git clone https://github.com/everglow1/hello-world.git

git remote

git 要求每个远程主机都必须有一个主机名， git remote 命令就用于管理主机名

git remote // 列出所有主机名

git remote -v // 查看远程主机的网址

克隆版本库的时候，所使用的远程主机会被命名为 origin ，如果你想用其它主机名， git clone -o

git remote show <主机名> // 查看该主机的详细信息

git remote add <主机名> <网址> // 添加远程主机

git remote rm <主机名> // 删除远程主机

git remote rename <原主机名> <新主机名> // 改远程主机名

git fetch

远程主机的版本库有了新的更新，需要将更新取回本地，需要用到 git fetch

git fetch <远程主机名> // 将远程主机的所有更新，取回本地

git fetch 通常用来查看其他人的进程，它对你的本地开发代码没有影响

git pull

git pull 作用： 取回远程主机某个分支的更新，再与本地指定分支合并。

例如
git pull <远程主机名> <远程分支名>:<本地分支名>

git pull origin next:master // 取回origin主机的next分支， 与本地的master分支合并
git pull origin next // 远程分支与当前分支合并

git push

git push 作用： 用于将本地分支的更新，推送到远程主机，格式与git pull相似

git push <远程主机名> <本地分支名>:<远程分支名>

git push origin master // 将本地的master分支推送到origin主机的master分支，假设后者不存在，将会被重建
