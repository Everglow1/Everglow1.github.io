前端的历史和趋势

发表于 2018-04-17 | 分类于 笔记 | 阅读次数: |

注：来源于阮一峰《全栈工程师》

什么是前端呢？

前端： 针对浏览器的开发，代码在浏览器运行
后端： 针对服务器的开发，代码在服务器运行

前后端不区分的时代

互联网发展的早期，前后端开发是一体的，前端代码是后端代码的一部分
当时的开法过程是这样的：

后端收到浏览器的请求 =》 生成静态页面 =》 发送到浏览器

在这个时代，网站开发，使用后端MVC的开发模式

Model(模型层) 提供，保存数据
Controller(控制层) 数据处理，实现业务逻辑
View(视图层) 展示数据，提供用户页面 （前端那个时候只是后端MVC中的V）

所以那个时候的前端工程师，只是模板工程师，负责编写模板，后端代码读取模板，替换变量，渲染出页面

Ajax技术

2004年，Gmail应用ajax技术， 2005年，谷歌地图应用ajax。
前面不再是后端的模板，介意独立得到各种数据。

Web2.0

Ajax促成了Web2.0的诞生， （Web1.0: 静态页面，纯内容展示 Web2.0: 动态网页，富交互，前端数据处理）
从这个时候起，前端变得复杂起来，对前端工程师的要求也越来越高。

前端的MVC模式

前端通过ajax得到数据，因此也有了处理数据的请求
前端代码也变得需要保存数据，处理数据，生成视图===》导致了前端MVC框架的生成

MVVM模式

用 View Model 代替Controller
Model
View
View-Model 简化的Controller，唯一的作用就是为View提供处理好的数据，不含其它逻辑

本质：view 绑定 view-model，视图与数据模型强耦合。数据的变化实时反映在 view 上，不需要手动处理。

有三大主流的MVVM模式的框架：谷歌的Angular, Facebook的React， 尤雨溪的Vue

SPA

SPA： Single page application
前端现在可以做到：读取数据，切换视图，用户交互

这意味着，网页其实是一个应用程序。 大概在10年左右，前端工程师从开发页面，变成了开发“前端应用”（跑在浏览器里面的程序）

前后端已经分离。

REST接口

前后端分离之后，它们之间通过接口通信，后端暴露出接口，前端使用后端提供的数据，后端的接口一般是REST形式，前后端通信的协议一般是http协议

Node

09年，Node项目诞生，它是服务器上的JavaScript运行环境

Node = JavaScript + 操作系统api

Node的诞生有重大意义： JavaScript 成为服务器脚本语言，与 Python 和 Ruby 一样，JavaScript 成为唯一的浏览器和服务器都支持的语言

前端工程师可以编写后端程序了

前端开发模式的根本改变

Node环境下开发，大量使用服务器端工具， 引入持续集成等软件工程的标准流程，开发完成后，编译成浏览器可以运行的脚本，放上CDN

全栈工程师

前端工程师正在转变成全栈工程师

一个人负责开发前端和后端，从数据库到ui的所有开发

全栈技能

  • 传统前端技能：HTML、JavaScript、CSS
  • 一门后端语言
  • 移动端开发：iOS / Android / HTML5
  • 其他技能：数据库、HTTP 等等

历史演变：前后端不分 -> 前后端分离 -> 全栈工程师
